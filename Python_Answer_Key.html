Exploratory Data Analysis (EDA) Report¶

Financial Fraud Detection – Python Analysis¶

Introduction¶

Exploratory Data Analysis (EDA) was performed on the credit card transaction dataset to understand its structure, detect anomalies, identify patterns, and determine key factors influencing fraudulent transactions. The objective is to support the development of a reliable fraud detection system for the financial industry.

Dataset Overview¶

The dataset contains:

  • 389,002 transactions
  • 22 variables
  • No missing values detected
  • Fraud rate ≈ 0.58% (highly imbalanced dataset)

The dataset includes customer information, transaction details, merchant data, and fraud labels.

Key Findings from EDA¶

Class Imbalance¶

Fraudulent transactions represent a very small proportion of total transactions, indicating a class imbalance problem. This is critical when building machine learning models.

Transaction Amount (amt)¶

  • Strongest correlation with fraud (correlation = 0.21).
  • Fraudulent transactions have significantly higher average amounts.
  • Distribution is highly right-skewed with extreme high-value outliers.

Transaction amount is a major predictive feature.

Transaction Category¶

Certain categories show higher fraud rates:

  • shopping_net
  • misc_net
  • grocery_pos

Online-based transactions exhibit higher fraud probability.

Category plays an important role in fraud detection.

Geographical Variables¶

  • Customer and merchant latitude/longitude show strong internal correlation.
  • City population is highly skewed.
  • No strong correlation with fraud detected.

Location alone is not a strong predictor but may help in advanced modeling.

Time-Based Trends¶

Monthly analysis showed fluctuations in transaction volume. Fraud occurrences follow transaction density but spike in certain periods, suggesting seasonal or behavioral trends.

Time features may improve model performance.

Outliers¶

  • Extreme values in transaction amount (up to $27,390).
  • Large variation in city population.
  • These may represent high-risk transactions.

Outlier handling may improve model stability.

Data Quality Checks¶

  • No duplicate transaction numbers.
  • No invalid latitude/longitude values.
  • No negative transaction amounts.
  • Age values mostly reasonable, though minor anomalies exist.

Overall data quality is strong.

Business Implications¶

Based on EDA:

  • High-value transactions require stricter fraud monitoring.
  • Online transaction categories should have enhanced risk controls.
  • Fraud detection models must handle severe class imbalance.
  • Feature engineering on time and transaction behavior can improve detection accuracy.

Conclusion¶

The EDA process provided critical insights into fraud behavior patterns. Transaction amount and category emerged as the most influential factors, while demographic and geographic features showed limited impact.

The dataset is clean and well-structured, making it suitable for predictive modeling. These insights form a strong foundation for developing a robust fraud detection system in the financial industry.
